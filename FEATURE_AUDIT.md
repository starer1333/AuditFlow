# AuditFlow 功能核对（基于当前仓库代码）

本文件用于核对“功能全景总结”与当前 `streamlit_app.py` 实现的一致性。

## 已实现（代码中可直接定位）

- 支持 6 类文件类型下拉选择：银行对账单、开户清单、销户清单/证明、企业信用报告、银行询证函回函、银行存款余额调节表。
- 五大痛点卡片展示（跨页合并、生僻汉字、中英文混排、水印印章、表格还原）。
- OCR 调用优先走 DeepSeek-OCR（SiliconFlow 网关），失败静默降级到模拟文本。
- 大模型分析优先走 Qwen2-VL-72B-Instruct，失败降级到模拟分析与 JSON。
- 文件类型/财务相关性校验：使用 warning 提示，不中断流程。
- API 状态提示：双真实 / OCR 真实 / LLM 真实 / 全模拟 四种分支。
- 账号 Luhn + BIN + 银行名交叉验证，并联动最终置信度。
- 五字段抽取与别名兼容：银行名称、账号、期末余额、对账期间、币种。
- 余额清洗：支持去货币符号与千分位后转浮点。
- 风险意见兜底：按余额负数/正数/缺失给出默认审计意见。
- 六类 Excel 底稿模板路由与下载。
- JSON 完整报告下载。
- 开篇故事、理论支撑、未来展望、数据安全、品牌页脚均已实现，且主要 expander 默认展开。

## 已补齐项（本次优化）

- 已新增本地 OCR 兜底分支：当云端 DeepSeek-OCR 不可用时，自动尝试本地 PaddleOCR（如环境已安装），再不行才降级模拟数据。
- 已新增临时文件清理逻辑：处理结束后无论成功/失败都会执行删除，减少数据残留风险。
- 已抽取统一 API 调用函数，减少 OCR 与大模型请求的重复代码，便于后续维护。

## 结论

当前仓库可实现你描述功能中的**绝大多数核心演示能力**，尤其是：上传识别、双模型降级、字段提取、账号校验、置信度联动、Excel/JSON 输出、品牌化展示。
